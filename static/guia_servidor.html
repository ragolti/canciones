Fase 1 — Ubuntu Server 💿
1

Instalar Ubuntu Server en la PC

Descargar la imagen, crear el USB y arrancar el instalador

⏱ Tiempo estimado: 1 a 2 horas
1

Bajar Ubuntu Server 24.04 LTS

Desde tu PC Windows, entrá al sitio oficial de Ubuntu y descargá la imagen ISO (gratis, ~2.5 GB):

https://ubuntu.com/download/server

Hacé clic en "Download Ubuntu Server 24.04 LTS". Guardalo en una carpeta fácil de encontrar.

2

Bajar Rufus (crea el USB booteable)

Rufus es un programa gratis y portable para Windows que convierte el archivo ISO en un USB de instalación:

https://rufus.ie

No necesita instalarse. Solo abrilo, insertá un USB de 4 GB o más y usalo como se explica en el paso 3.

3

Crear el USB booteable con Rufus

Con el USB insertado, abrí Rufus y seguí estos pasos:

1. Device: elegí tu USB
2. Boot selection: clic en "SELECT" → buscá el archivo .iso descargado
3. Partition scheme: dejalo en "GPT"
4. Clic en START → confirmar que se va a borrar el USB

⚠️ El USB se va a formatear. Asegurate de que no tenga archivos importantes.
4

Conectar la PC servidor por cable de red

Antes de instalar, conectá la PC del servidor al router MikroTik por cable ethernet (no WiFi). Esto es importante para que el instalador configure la red automáticamente y te asigne una IP fija más adelante.

5

Arrancar desde el USB

Con el USB insertado en la PC servidor, encendela y entrá al menú de arranque (Boot Menu). Dependiendo de la placa, la tecla puede ser:

F2, F8, F10, F12 o Del — probalas ni bien empieza a encender.

Una vez en el menú, elegí el USB como dispositivo de arranque.

6

Seguir el instalador de Ubuntu Server

El instalador es modo texto pero muy sencillo. Lo más importante:

 • Idioma y teclado: elegí español si querés
 • Tipo de instalación: "Install Ubuntu Server"
 • Red: debería detectar el cable solo (IP por DHCP)
 • Disco: "Use an entire disk" y confirmá
 • Nombre del servidor: ponele algo como adoremos-server
 • Usuario y contraseña: anotálos, los vas a necesitar siempre
 • OpenSSH: ✅ marcá la opción "Install OpenSSH server"

📌 El OpenSSH es clave: te permite manejarlo desde tu PC Windows sin necesitar monitor ni teclado en el servidor.
7

Reiniciar y sacar el USB

Al terminar, el instalador te pide que saques el USB y presiones Enter. El servidor va a reiniciar y quedar listo con Ubuntu instalado.

Vas a ver una pantalla de login en modo texto. Eso es normal y correcto.

Fase 2 — App Flask 🐍
2

Configurar la app en el servidor

Instalar Python, Gunicorn y Nginx, y dejar la app corriendo como servicio

⏱ Tiempo estimado: 1 hora
Todo lo que sigue se hace por SSH desde tu PC Windows. Abrí PowerShell o el símbolo del sistema y conectate con:
ssh usuario@IP-LOCAL-DEL-SERVIDOR Para saber la IP local del servidor, mirá el panel del MikroTik o escribí en el servidor: ip a
1

Actualizar el sistema e instalar dependencias

sudo apt update && sudo apt upgrade -y
sudo apt install -y python3 python3-pip python3-venv nginx git
2

Copiar el código de la app al servidor

La forma más simple es clonar desde GitHub (si el repo es privado, necesitás un token de acceso):

cd /home/USUARIO
git clone https://github.com/TU-USUARIO/canciones.git
cd canciones

O podés copiar los archivos con scp desde Windows.

3

Crear entorno virtual e instalar dependencias

python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt gunicorn
4

Crear el servicio con systemd

Esto hace que la app arranque sola cuando el servidor se reinicia:

sudo nano /etc/systemd/system/canciones.service

Pegá este contenido (reemplazá USUARIO con tu nombre de usuario):

[Unit]
Description=App Canciones Flask
After=network.target

[Service]
User=USUARIO
WorkingDirectory=/home/USUARIO/canciones
ExecStart=/home/USUARIO/canciones/.venv/bin/gunicorn -w 2 -b 127.0.0.1:5000 app:app
Restart=always

[Install]
WantedBy=multi-user.target sudo systemctl enable canciones
sudo systemctl start canciones
sudo systemctl status canciones
5

Configurar Nginx como proxy

Nginx recibe las visitas del puerto 80/443 y las pasa a Gunicorn:

sudo nano /etc/nginx/sites-available/canciones server {
    listen 80;
    server_name TU-DOMINIO.COM;

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
} sudo ln -s /etc/nginx/sites-available/canciones /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
Fase 3 — Acceso desde internet 🌐
3

Publicar en internet

MikroTik, DDNS gratuito y HTTPS con Let's Encrypt

⏱ Tiempo estimado: 1 hora
1

Asignar IP fija al servidor en el MikroTik

En el panel del MikroTik (usualmente en 192.168.1.1), buscá la sección IP > DHCP Server > Leases. Encontrá la PC servidor y hacé clic en "Make Static". Así siempre va a tener la misma IP local.

💡 Anotá la IP local del servidor (ej: 192.168.1.100). La vas a usar en el siguiente paso.
2

Abrir puertos 80 y 443 en el MikroTik

En el MikroTik: IP > Firewall > NAT > + (Agregar)

Hacé esto DOS veces (una para el puerto 80, otra para el 443):

 • Chain: dstnat
 • Protocol: tcp
 • Dst. Port: 80 (o 443)
 • Action: dst-nat
 • To Addresses: 192.168.1.100 (IP del servidor)
 • To Ports: 80 (o 443)

3

Activar DDNS en el MikroTik (dominio gratuito)

El MikroTik tiene DDNS gratis integrado. En el panel: IP > Cloud

Activá "DDNS Enabled" y copiá el dominio que te asigna, algo como:
xxxxxxxx.sn.mynetname.net

Ese dominio va a apuntar siempre a tu IP, aunque cambie.

4

Actualizar el dominio en Nginx

Editá el archivo de Nginx que creaste antes y reemplazá TU-DOMINIO.COM con el dominio DDNS del paso anterior:

sudo nano /etc/nginx/sites-available/canciones sudo systemctl reload nginx
5

Instalar HTTPS gratuito con Let's Encrypt

Certbot configura HTTPS automáticamente y renueva el certificado solo:

sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d TU-DOMINIO.DDNS.net

Seguí las instrucciones (ponés tu email, aceptás términos). Al terminar, tu app ya tiene el candado 🔒 y funciona con https://.

✅ Certbot renueva el certificado automáticamente. No hay que hacer nada cada 90 días.
6

Probar que todo funciona

Desde cualquier dispositivo con internet (incluso el celular con datos, no WiFi de tu casa), abrí:

https://TU-DOMINIO.DDNS.net

Si ves la app, ¡el servidor propio está funcionando! La respuesta debería ser instantánea comparada con Render.
